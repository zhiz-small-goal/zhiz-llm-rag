## 4. 审查要点（按优先级）
### 4.1 文档清晰度
- 入口可发现：README / docs/INDEX / reference 是否有跳转
- 操作步骤：是否可执行；是否包含验收信号
- 版本/日期：是否明确

### 4.2 代码质量
- 退出码：0/2/3 契约是否遵循
- 诊断定位：是否使用 file:line:col
- 测试/门禁：关键不变量是否覆盖

### 4.3 可复用性
- SSOT：是否避免双写与漂移
- 入口形式：console_scripts / python -m / wrapper

### 4.4 安全性
- secrets：是否存在明文密钥/令牌
- 依赖：新增依赖是否有动机与替代

### 4.5 报告可读性
- summary：是否有一眼可读的结论与关键计数
- JSON：字段稳定且可被机器消费
