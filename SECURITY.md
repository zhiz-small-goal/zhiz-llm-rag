# Security Policy

## 报告安全漏洞（请勿创建公开 Issue）

请通过 GitHub 仓库页面的 **Security** 入口提交私密报告（或遵循仓库 security policy 页面指引）。

## 支持范围

- 默认仅支持 **main 分支最新提交**（不承诺为旧提交/旧版本回溯修复）
- 若需要补丁，请在报告中提供：影响范围、复现步骤、版本/commit、建议修复方向（可选）

## 披露

维护者将根据影响程度决定修复与披露节奏，不承诺时间线。
